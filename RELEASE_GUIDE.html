1 One-time setup do once

Install these tools on your build machine:

  • Flutter SDK — already installed (used for flutter run -d windows).
  • Visual Studio 2022 with the Desktop development with C++ workload — required by Flutter to build Windows apps.
  • Inno Setup 6 — download from jrsoftware.org/isinfo.php and install with default options.
Already done in your repo The installer/OptiSana.iss script has been created. You do not need to recreate it.

2 Bump the version every release

Update the version in two places so the exe metadata, the installer filename, and the uninstall entry all agree.

a. pubspec.yaml

version: 1.0.1+2   # was 1.0.0+1
  • 1.0.1 — user-visible version (Major.Minor.Patch).
  • +2 — internal build number. Increment every release.

b. installer/OptiSana.iss

#define MyAppVersion "1.0.1"
Do not change MyAppId The GUID in AppId identifies the installed app to Windows. Keep it the same so clients get an upgrade instead of a second parallel install.

3 Build the Windows release every release

Run these from the project root in PowerShell:

flutter clean
flutter pub get
flutter build windows --release

Output lands in:

build\windows\x64\runner\Release\
├── OptiSana.exe
├── flutter_windows.dll
├── audioplayers_windows_plugin.dll
├── sqlite3.dll            (and other plugin DLLs)
└── data\
    ├── icudtl.dat
    └── flutter_assets\    (translations, sounds, images)
Why flutter clean? It forces CMake to regenerate, which is only strictly needed the first time after renaming the exe. You can skip it on subsequent releases if nothing in windows/ changed — but running it is always safe.

4 Compile the installer every release

Two equivalent ways to compile the .iss script:

Option A — command line (recommended)

& "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer\OptiSana.iss

Option B — GUI

  1. Open installer/OptiSana.iss in Inno Setup Compiler.
  2. Press F9 (or Build → Compile).

Result:

build\installer\OptiSana-Setup-1.0.1.exe
That single .exe is everything your clients need. It contains the app, all DLLs, and the data\ folder compressed with LZMA2.

5 Ship it every release

Deliver OptiSana-Setup-1.0.1.exe by email, USB, WeTransfer, or a shared drive. The client experience:

  1. Double-click the installer.
  2. Windows SmartScreen may say "Windows protected your PC" → click More info → Run anyway. (This only disappears with a paid code-signing certificate.)
  3. UAC prompts for admin — approve.
  4. Next → Next → Install → Finish.
  5. Installed to C:\Program Files\OptiSana\ with a Start Menu shortcut (and desktop shortcut if they ticked the box).
Upgrades are automatic Running a newer OptiSana-Setup-X.Y.Z.exe over an existing install replaces the files in place. User data is preserved:
  • Database: %UserProfile%\Documents\OpticaSana.db
  • Preferences (password, font size, sound): HKCU\Software\Naseem Srour\OptiSana

7 Notes & troubleshooting

SmartScreen warning

Without code-signing, clients see "Windows protected your PC" once. They click More info → Run anyway. To remove this permanently, purchase a code-signing certificate (~$100–250/year) and run signtool sign on both OptiSana.exe and OptiSana-Setup-X.Y.Z.exe.

Visual C++ Redistributable

The app depends on the MSVC runtime. It's pre-installed on all modern Windows 10/11, but if a client ever hits a "VCRUNTIME140.dll missing" error, point them to aka.ms/vs/17/release/vc_redist.x64.exe.

Rolling back an install

The installer registers a proper uninstaller. Clients can remove the app via Settings → Apps → OptiSana → Uninstall. Their database and preferences are kept.

Replacing the app icon

Drop a new 256×256 multi-resolution .ico at windows/runner/resources/app_icon.ico and rebuild. The exe, installer, and Start Menu shortcut all update automatically.

If the build fails

  • CMake Error about MSVC → open Visual Studio Installer and make sure Desktop development with C++ is checked.
  • Target of URI doesn't exist → run flutter pub get.
  • Old exe name lingers (Optisana.exe) → run flutter clean then rebuild.
